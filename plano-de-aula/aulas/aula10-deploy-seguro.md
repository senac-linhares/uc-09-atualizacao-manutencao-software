# Aula 10 — Deploy seguro, rollback e feature flags

## 🎯 Objetivo
Conhecer práticas para publicar mudanças com menos risco.

## Conteúdos
- checklist de deploy;
- rollback;
- feature flag;
- validação pós-publicação.

## Explicação simples
Deploy seguro é publicar com cuidado. Se algo der errado, a equipe precisa saber voltar rapidamente para uma versão estável.

## Atividade rápida
Montar um plano de rollback para uma atualização simples.

## Fechamento
Publicar rápido é bom; publicar com segurança é melhor ainda.
