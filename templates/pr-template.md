# Template de Pull Request

## O que foi feito?
Descreva a correção, melhoria ou atualização.

## Por que foi necessário?
Explique o problema ou a necessidade atendida.

## Como foi validado?
- [ ] teste manual
- [ ] teste automatizado
- [ ] revisão de código

## Riscos conhecidos
Informe possíveis impactos.

## Checklist
- [ ] código revisado
- [ ] documentação atualizada
- [ ] testes executados
- [ ] plano de rollback definido
