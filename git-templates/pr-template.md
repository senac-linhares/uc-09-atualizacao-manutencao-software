# Template de Pull Request

## 🔗 Contexto e Rastreabilidade
- **Issue relacionada:** #00
- **Tipo de alteração:** 
    - [ ] 🔴 Bugfix (correção de erro)
    - [ ] 🔵 Feature (nova funcionalidade)
    - [ ] 🟡 Refatoração (melhoria de código sem mudar a função)
    - [ ] 🟣 Documentação / Outros

## 📝 O que foi feito?
> Descreva de forma técnica as mudanças realizadas. Use listas se necessário.

*(Se for uma Feature, utilize o formato de User Story: Como [usuário], quero [ação] para [valor])*

## ❓ Por que foi necessário?
> Explique o problema ou a necessidade de negócio que motivou esta alteração.

## ⚠️ Mudanças Críticas (Breaking Changes)
- [ ] **Não** possui mudanças que quebram compatibilidade.
- [ ] **Sim**, possui mudanças críticas. (Descreva abaixo: ex: mudou nome de tabela no banco).

## 🖼️ Evidências Visuais
> Se for uma mudança visual, anexe prints ou GIFs comparando o "Antes" e "Depois".

## ✅ Checklist de Qualidade
- [ ] O código segue o guia de estilo do projeto.
- [ ] Realizei testes manuais no ambiente local.
- [ ] A documentação/Wiki foi atualizada (se necessário).
- [ ] Não há "logs" de debug esquecidos no código.

## 🧪 Como testar?
1. Faça o checkout para a branch `nome-da-branch`.
2. Execute o comando `...`.
3. Verifique se o comportamento `X` acontece.
